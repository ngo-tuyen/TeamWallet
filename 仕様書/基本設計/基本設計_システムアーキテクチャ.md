## 4. APIエンドポイント

### 4.1 認証 (Auth)

```
POST   /auth/register           # ユーザー登録
POST   /auth/login              # ログイン
```

### 4.2 チーム (Teams)

```
POST   /api/teams               # チーム作成
GET    /api/teams               # チーム一覧取得
GET    /api/teams/:teamId       # チーム詳細取得
PUT    /api/teams/:teamId       # チーム更新
DELETE /api/teams/:teamId       # チーム削除
```

### 4.3 メンバー (Members)

```
GET    /api/teams/:teamId/members                      # メンバー一覧
PUT    /api/teams/:teamId/members/:memberId/role       # ロール変更
DELETE /api/teams/:teamId/members/:memberId            # メンバー削除
```

### 4.4 招待 (Invitations)

```
GET    /api/invitations/search?username=xxx            # ユーザー検索
POST   /api/invitations/teams/:teamId/invite           # 招待送信
GET    /api/invitations/my-invitations                 # 招待一覧取得
POST   /api/invitations/:invitationId/accept           # 招待受け入れ
POST   /api/invitations/:invitationId/reject           # 招待拒否
```

### 4.5 カテゴリ (Categories)

```
GET    /api/teams/:teamId/categories                   # カテゴリ一覧
POST   /api/teams/:teamId/categories                   # カテゴリ追加
DELETE /api/teams/:teamId/categories/:categoryId       # カテゴリ削除
```

### 4.6 収入 (Incomes)

```
GET    /api/teams/:teamId/incomes?month=YYYY-MM       # 収入一覧
POST   /api/teams/:teamId/incomes                      # 収入追加
PUT    /api/teams/:teamId/incomes/:incomeId           # 収入更新
DELETE /api/teams/:teamId/incomes/:incomeId           # 収入削除
```

### 4.7 支出 (Expenses)

```
GET    /api/teams/:teamId/expenses?month=YYYY-MM&categoryId=X  # 支出一覧
POST   /api/teams/:teamId/expenses                     # 支出追加
PUT    /api/teams/:teamId/expenses/:expenseId         # 支出更新
DELETE /api/teams/:teamId/expenses/:expenseId         # 支出削除
```

### 4.8 サマリー (Dashboard)

```
GET    /api/teams/:teamId/summary?month=YYYY-MM       # 月別サマリー
```

### 4.9 管理者 (Admin - Super Admin Only)

```
GET    /api/admin/stats                                # システム統計
GET    /api/admin/teams                                # 全チーム一覧
DELETE /api/admin/teams/:teamId                        # チーム削除
```

## 5. 認証・認可設計

### 5.1 認証フロー

```
[ログイン]
  ↓
POST /auth/login { username, password }
  ↓
[ユーザー認証]
  ├─ 失敗 → 401 Unauthorized
  └─ 成功
    ↓
[JWT生成]
  ↓
[トークン返却]
  ↓
[ローカルストレージに保存]
  ↓
[API呼び出し時にAuthorizationヘッダーに付与]
```

### 5.2 ロールベースアクセス制御 (RBAC)

```
| 機能        | Admin | Manager | User |
|-------------|-------|---------|------|
| チーム表示   |  ✅  |    ✅   |  ✅ |
| チーム作成   |  ✅  |    ❌   |  ❌ |
| チーム削除   |  ✅  |    ❌   |  ❌ |
| メンバー招待 |  ✅  |    ❌   |  ❌ |
| ロール変更   |  ✅  |    ❌   |  ❌ |
| メンバー削除 |  ✅  |    ❌   |  ❌ |
| 収入追加     |  ✅  |    ✅   |  ❌ |
| 支出追加     |  ✅  |    ✅   |  ❌ |
| 収支表示     |  ✅  |    ✅   |  ✅ |
| 招待管理     |  ✅  |    ✅   |  ✅ |
```

## 6. エラーハンドリング

### 6.1 HTTPステータスコード

```
200 OK              # リクエスト成功
201 Created         # リソース作成成功
400 Bad Request     # 入力エラー (バリデーション失敗等)
401 Unauthorized    # 認証エラー (ログインしていない)
403 Forbidden       # 権限エラー (アクセス権限なし)
404 Not Found       # リソース不存在
409 Conflict        # リソース重複 (同じユーザー名等)
500 Server Error    # サーバーエラー
```

### 6.2 エラーレスポンス形式

```json
{
  "success": false,
  "message": "エラーメッセージ",
  "code": "ERROR_CODE",
  "data": null
}
```
