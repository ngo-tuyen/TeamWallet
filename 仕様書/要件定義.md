## 1. プロジェクト概要

### 1.1 プロジェクト情報

| 項目 | 説明 |
|------|------|
| **プロジェクト名** | Team Wallet |
| **説明** | チーム・グループで収支を共有・管理するWebアプリケーション |
| **開発言語** | React (Frontend), Node.js/Express (Backend), MySQL (Database) |
| **プロジェクトタイプ** | 個人プロジェクト（学習目的）|
| **主目的** | 上流工程の学習（要件定義 → 設計 → コーディング→デプロイ）

### 1.2 プロジェクトの目的

- **背景**: 自分が現在、ベトナムのサッカーチームの収支管理を Excel で行っている
- **課題**: グループメンバー間で収支情報を共有・管理する仕組みがない
- **解決策**: クラウドベースのチーム収支管理アプリケーション
- **メリット**: 誰でも簡単にグループの収支を追跡できる

## 2. 機能要件

### 2.1 認証・ユーザー管理

#### 2.1.1 ユーザー登録
```
エンドポイント: POST /auth/register
入力:
  - username: VARCHAR(255), UNIQUE, NOT NULL, 6文字以上
  - password: VARCHAR(255), NOT NULL, 6文字以上
  - full_name: VARCHAR(255), NOT NULL
出力:
  - success: boolean
  - message: string
```

**バリデーション:**
- ユーザー名: 6文字以上、重複チェック
- パスワード: 6文字以上
- フルネーム: 空白チェック

#### 2.1.2 ログイン
```
エンドポイント: POST /auth/login
入力:
  - username: string
  - password: string
出力:
  - token: JWT (12h有効)
  - user: { user_id, username, full_name }
```

#### 2.1.3 スーパー管理者
```
ユーザー名: SUPER_admIn
パスワード: SUPER_admIn_121314
権限: システム全体の管理
```

---

### 2.2 チーム管理

#### 2.2.1 チーム作成
```
エンドポイント: POST /api/teams
入力:
  - team_name: VARCHAR(255), NOT NULL
  - description: TEXT, NULL
出力:
  - team_id: INT
  - team_name: string
  - admin_id: INT (自動的に現在ユーザー)
```

**ルール:**
- チーム作成者は自動的に **admin** ロール
- チーム作成後、メンバー招待可能

#### 2.2.2 チーム詳細表示
```
エンドポイント: GET /api/teams/:teamId
レスポンス:
  - team_id, team_name, description
  - members: [{ user_id, username, role }]
  - created_at, updated_at
```

#### 2.2.3 チーム削除
```
エンドポイント: DELETE /api/teams/:teamId
権限: Admin のみ
効果: ON DELETE CASCADE で関連データも削除
```

---

### 2.3 メンバー管理 & 招待システム

#### 2.3.1 メンバー招待
```
エンドポイント: POST /api/invitations/teams/:teamId/invite
入力:
  - user_id: INT
  - role: ENUM('manager', 'user')  ※ admin は招待不可
出力:
  - invitation_id: INT
  - status: 'pending'
```

**ルール:**
- Admin のみが招待可能
- 招待可能なロール: User, Manager のみ
- Admin ロールは招待不可 (チーム作成者のみ)

#### 2.3.2 招待の受け入れ/拒否
```
受け入れ:
  エンドポイント: POST /api/invitations/:invitationId/accept
  効果: team_members に追加、invitation を削除

拒否:
  エンドポイント: POST /api/invitations/:invitationId/reject
  効果: invitation を削除
```

#### 2.3.3 ロール管理
```
ロール: admin, manager, user

| ロール | 権限 |
|--------|------|
| admin | メンバー管理・ロール変更・チーム削除 (チーム作成者のみ) |
| manager | 収入・支出の追加・編集 |
| user | 閲覧のみ |
```

**ロール変更ルール:**
- Admin のみが変更可能
- チーム作成者は自分のロール変更不可
- Non-Admin ユーザーは変更ボタン無効化

#### 2.3.4 メンバー削除
```
エンドポイント: DELETE /api/teams/:teamId/members/:memberId
権限: Admin のみ
```

---

### 2.4 支出カテゴリ管理

#### 2.4.1 カテゴリ作成
```
エンドポイント: POST /api/teams/:teamId/categories
入力:
  - category_name: VARCHAR(255), NOT NULL
出力:
  - category_id: INT
  - team_id: INT
  - category_name: string
```

**ルール:**
- チームごとに独立
- チーム内でカテゴリ名は一意

#### 2.4.2 カテゴリ削除
```
エンドポイント: DELETE /api/teams/:teamId/categories/:categoryId
権限: Manager以上
効果: ON DELETE CASCADE で関連する支出も削除
```

---

### 2.5 収入管理

#### 2.5.1 収入登録
```
エンドポイント: POST /api/teams/:teamId/incomes
入力:
  - amount: DECIMAL(10, 2), NOT NULL, >= 1
  - income_date: DATE, NOT NULL, 過去日付可能
  - memo: VARCHAR(255), NULL
出力:
  - income_id: INT
  - team_id, amount, income_date, memo
  - created_at, updated_at
権限: Manager以上
```

#### 2.5.2 収入表示
```
エンドポイント: GET /api/teams/:teamId/incomes?month=2025-03
レスポンス:
  - incomes: [{income_id, amount, income_date, memo}]
  - 総収入, 件数, 平均値
```

#### 2.5.3 収入更新・削除
```
更新: PUT /api/teams/:teamId/incomes/:incomeId
削除: DELETE /api/teams/:teamId/incomes/:incomeId
権限: Manager以上
```

---

### 2.6 支出管理

#### 2.6.1 支出登録
```
エンドポイント: POST /api/teams/:teamId/expenses
入力:
  - category_id: INT, NOT NULL
  - amount: DECIMAL(10, 2), NOT NULL, >= 1
  - expense_date: DATE, NOT NULL, 過去日付可能
  - memo: VARCHAR(255), NULL
出力:
  - expense_id: INT
  - team_id, category_id, amount, expense_date, memo
  - created_at, updated_at
権限: Manager以上
```

#### 2.6.2 支出表示
```
エンドポイント: GET /api/teams/:teamId/expenses?month=2025-03&category=1
レスポンス:
  - expenses: [{expense_id, category_id, amount, expense_date, memo}]
  - 総支出, 件数, 平均値
```

#### 2.6.3 支出更新・削除
```
更新: PUT /api/teams/:teamId/expenses/:expenseId
削除: DELETE /api/teams/:teamId/expenses/:expenseId
権限: Manager以上
```

---

### 2.7 月別サマリー

#### 2.7.1 サマリー表示
```
エンドポイント: GET /api/teams/:teamId/summary?month=2025-03
レスポンス:
  - month: "2025-03"
  - total_income: DECIMAL(10, 2)
  - total_expense: DECIMAL(10, 2)
  - balance: DECIMAL(10, 2) = total_income - total_expense
  - avg_income: DECIMAL(10, 2)
  - avg_expense: DECIMAL(10, 2)
  - income_count: INT
  - expense_count: INT
```

**表示内容:**
- 総収入・総支出・残額
- 平均値 (収入・支出ごと)
- 黒字/赤字判定

## 3. 非機能要件

### 3.1 セキュリティ
- **認証**: JWT (HS256)
- **パスワード**: bcryptjs (ラウンド数: 10)
- **SQL対策**: プリペアドステートメント使用

### 3.2 パフォーマンス
- **応答時間**: 2秒以内を目標
- **同時接続**: 100ユーザー以上対応
- **キャッシング**: API結果をローカルストレージにキャッシュ

### 3.3 保守性
- **API設計**: REST準拠
- **コード構成**: モジュール化
- **ドキュメント**: 完備

## 4. データベース要件

### 4.1 テーブル一覧

| テーブル名 | 説明 | 主要カラム |
|-----------|------|----------|
| users | ユーザー情報 | user_id (PK), username (UQ), password, full_name |
| teams | チーム情報 | team_id (PK), team_name, admin_id (FK), description |
| team_members | チームメンバー | member_id (PK), user_id (FK), team_id (FK), role |
| invitations | 招待情報 | invitation_id (PK), team_id (FK), user_id (FK), role, status |
| categories | 支出カテゴリ | category_id (PK), team_id (FK), category_name |
| incomes | 収入情報 | income_id (PK), team_id (FK), amount, income_date |
| expenses | 支出情報 | expense_id (PK), team_id (FK), category_id (FK), amount, expense_date |
